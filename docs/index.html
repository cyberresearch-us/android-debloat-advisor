Android Package Advisor

Remove bloatware from Android phones — safely, with suggestions you review before anything runs.   Let an AI do it  ·  Parental controls

Safe and reversible. Commands generated here use pm uninstall -k --user 0 which disables an app for your profile but keeps the file on the phone. Restore any time with adb shell cmd package install-existing <package>.

How it works

You need a computer, a USB cable, and about 10 minutes. Follow the steps below, or paste our prompt into an AI tool and let it do the technical parts.

1
Turn on Developer Mode on the phone

This unlocks USB debugging so your computer can talk to the phone.

Show me how (Samsung / most Android)
  1. Open Settings.
  2. Scroll down, tap About phone.
  3. Tap Software information.
  4. Find Build number and tap it 7 times quickly.
  5. Enter your PIN if asked.
  6. You'll see: "Developer mode has been enabled."
  7. Go back to Settings — Developer options now appears near the bottom.
  8. Open it, scroll to Debugging, turn on USB debugging, tap OK.
2
Install ADB on your computer

ADB (Android Debug Bridge) is a small tool from Google — no Android Studio needed.

Windows
  1. Download Platform Tools ZIP from Google.
  2. Extract it to a folder like C:\platform-tools.
  3. Open that folder, hold Shift, right-click blank space, choose Open PowerShell window here.
  4. Type: .\adb.exe --version — you should see "Android Debug Bridge version…"
macOS
  1. Install via Homebrew: brew install android-platform-tools
  2. Or download the ZIP from the same Google link above and extract.
  3. Open Terminal, run: adb version
Linux
  1. sudo apt install adb (Debian/Ubuntu) or sudo dnf install android-tools (Fedora)
  2. Or download from Google and extract.
  3. Run: adb version
3
Connect the phone and get the package list

Plug in the USB cable. Accept the "Allow USB debugging?" prompt on the phone if it appears.

Show me the commands

First, verify the phone is connected:

adb devices

You should see your device listed as device (not unauthorized).

Then run this to get the package list:

adb shell pm list packages -f

This prints hundreds of lines — that is normal. To copy it all to your clipboard:

Windows PowerShell:

.\adb.exe shell pm list packages -f | Set-Clipboard

macOS Terminal:

adb shell pm list packages -f | pbcopy

Linux:

adb shell pm list packages -f | xclip -sel clip

Now paste it into the box below.

What to do next

  1. Make sure the phone is still connected: adb devices
  2. Paste the commands above into your terminal (PowerShell, Terminal, or Command Prompt).
  3. Each command prints Success or Failure.
  4. Test the phone — make a call, open the camera, send a text.
  5. If something broke, restore it immediately: adb shell cmd package install-existing com.example.package
  6. When you're done, turn off USB debugging: Settings → Developer options → USB debugging → OFF.

Parental controls

Removing apps is a good start, but it does not block the internet or prevent new installs. Pair this tool with Google Family Link for app approval, web filtering, and screen time.

  1. Set up a supervised child Google account in your family group.
  2. Install Family Link on your phone and the child's device.
  3. Require approval for new Play Store installs.
  4. Turn on web and content filters (options vary by region).
  5. Review Install unknown apps permissions on the child device.

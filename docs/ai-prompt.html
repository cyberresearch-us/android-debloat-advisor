Let an AI do it

Copy the prompt below, paste it into Cursor, Windsurf, Claude Code, or any AI tool with terminal access. The AI handles every technical step. You review and approve.

What you need: A USB cable, ADB on your computer, and USB debugging turned on. Not sure how? The prompt walks the AI through helping you set those up too.
I have an Android phone and I need your help removing bloatware. Follow the steps below one at a time. At each STOP gate, you MUST pause, show me what you found, and wait for me to say "continue" before moving on. --- STEP 1 — Help me enable USB debugging Tell me to do these things on the phone: 1. Open Settings. 2. Tap About phone. 3. Tap Software information. 4. Tap Build number 7 times. Enter PIN if asked. 5. Go back to Settings. Open Developer options. 6. Turn on USB debugging. Tap OK. 7. Plug the phone into the computer with a USB cable. 8. If the phone asks "Allow USB debugging?" — tap Allow. STOP. Ask me: "Is USB debugging on? Is the phone plugged in?" Wait for me to say "continue". --- STEP 2 — Check if ADB is installed First, find out if this is Windows, macOS, or Linux. On Windows PowerShell, always use .\adb.exe (with .\), never bare "adb". On macOS/Linux, use "adb". Admin rights are NOT needed for any of this. Run: adb version (or .\adb.exe version on Windows PowerShell) If it prints a version number, skip to Step 3. If it says "not recognized" or "command not found", tell me: - Windows: download Platform Tools from https://developer.android.com/studio/releases/platform-tools, extract to C:\platform-tools, then run: .\adb.exe version from that folder - macOS: run: brew install android-platform-tools - Linux: run: sudo apt install adb STOP. Show me the version output. Wait for me to say "continue". --- STEP 3 — Check phone connection Run: adb devices You should see one line with a serial number and the word "device". If it says "unauthorized", tell me to tap Allow on the phone. If nothing is listed, tell me to try a different USB cable. STOP. Tell me: what device is connected? Show the serial and status. Wait for me to say "continue". --- STEP 4 — Get the list of installed apps Run: adb shell pm list packages -f Count how many lines start with "package:". STOP. Tell me: "Found [N] packages. Ready to sort them." Wait for me to say "continue". --- STEP 5 — Sort packages into three groups Go through every package. Put each one into exactly one group: REMOVE — Bloatware the user probably does not want: Facebook, Instagram, TikTok, Microsoft stubs, Samsung Game tools, AR/VR, Bixby, Galaxy Store, carrier apps (Verizon, AT&T, T-Mobile), Netflix stubs, ANT+ radio, Samsung TV Plus, HeyTap/OPLUS cloud. KEEP — Removing these will break the phone: Launcher, Dialer, In-call UI, IMS, Play Services, Google Services Framework, WebView, Permission Controller, Settings, SystemUI, Telephony, Package Installer, Contacts Provider, Media Provider. REVIEW — Everything else. I will decide. Show me three tables: REMOVE, KEEP, REVIEW. For each package show: package name, short description, your group. STOP. Show me: - Counts: how many REMOVE, KEEP, REVIEW. - The full REMOVE list so I can check it. - The full KEEP list so I can verify. - The REVIEW list so I can decide. Do NOT continue until I say "approved" or give you changes. --- STEP 6 — Show the commands (do NOT run yet) For each package I approved for removal, write the uninstall command: - If the path contains /data/app: adb shell pm uninstall --user 0 PACKAGE - All others: adb shell pm uninstall -k --user 0 PACKAGE STOP. Show me every command. Tell me: "These are [N] commands. Say execute to run them." Do NOT run anything. Wait for me to say "execute". --- STEP 7 — Run the commands Run only the commands I approved. Show the result of each: Success or Failure. After all are done, show a summary: how many succeeded, how many failed. STOP. Show me the summary. Wait for me to say "continue". --- STEP 8 — Test the phone Tell me to test these things: - Make a phone call. - Open the camera. - Send a text message. - Check notifications. - Open any app I said to keep (Maps, email, etc.). Remind me: to restore any app, run: adb shell cmd package install-existing PACKAGE_NAME STOP. Ask me: "Did everything work? Tell me if anything is broken." Wait for my answer. --- STEP 9 — Fix anything broken If I report a problem: 1. Tell me which removed package probably caused it. 2. Show me the restore command. 3. Wait for me to approve. 4. Run it. 5. Ask me to test again. 6. Repeat until fixed. If I say everything works, go to Step 10. --- STEP 10 — Clean up Tell me to: 1. Open Settings > Developer options. 2. Turn OFF USB debugging. 3. Disconnect the USB cable. 4. Restart the phone. We are done. --- RULES (follow at all times): - NEVER skip a STOP gate. Always wait for my response. - NEVER run an uninstall command without my approval. - NEVER remove com.google.android.webview. It is NOT a browser. - NEVER remove the launcher, dialer, or messaging app. - If unsure about a package, put it in REVIEW, not REMOVE. - If I say something is broken, help me restore it immediately.

What to expect

The AI stops at safety gates. Nothing runs without your say-so.

  1. Steps 1–2 — Setup help: USB debugging and ADB install.
  2. Step 3 — AI confirms the phone is connected.
  3. Step 5 — AI shows Remove / Keep / Review lists. You pick what gets removed.
  4. Step 6 — AI shows every command before running. You say "execute".
  5. Step 8 — You test the phone. AI helps restore anything broken.

About 10 minutes start to finish.

Device Setup Guide

This guide walks you through every step: enabling Developer Options on the phone, installing ADB on your computer, connecting the two, and exporting the package list into the advisor.

What ADB can do here: The advisor generates pm uninstall -k --user 0 commands. These disable apps for user 0 but leave the APK on the system image — they are reversible. You run the commands; the advisor never connects to your phone directly.

Part 1 — Enable Developer Options on the phone

These steps cover Samsung One UI (Android 10–14). Other Android phones are similar; the menus may use slightly different names.

  1. Open Settings.
  2. Scroll to the bottom and tap About phone.
  3. Tap Software information.
  4. Find Build number and tap it 7 times in quick succession.
  5. You may be asked for your PIN or pattern.
  6. You will see a toast: "Developer mode has been enabled".
  7. Go back to the main Settings screen — you will now see Developer options near the bottom (sometimes under General management depending on One UI version).

Part 2 — Enable USB Debugging

  1. Open Settings → Developer options.
  2. Make sure the toggle at the top is ON.
  3. Scroll down to the Debugging section.
  4. Turn on USB debugging.
  5. Read the warning and tap OK.
Developer options and USB debugging increase attack surface slightly. Turn USB debugging off again when you are finished.

Part 3 — Install Android Platform Tools on your computer

Platform Tools is a small download from Google that includes adb.exe (Windows) or adb (macOS/Linux). No Android Studio needed.

Windows

  1. Download from developer.android.com/studio/releases/platform-tools.
  2. Extract the ZIP to a folder you can find easily, e.g. C:\platform-tools.
  3. Open File Explorer, navigate to that folder, hold Shift and right-click blank space, choose Open PowerShell window here.
  4. Confirm ADB is present:
    .\adb.exe --version
    You should see a line like Android Debug Bridge version 1.0.41.

macOS

  1. Download and extract from the same Google link above, or install via Homebrew:
    brew install android-platform-tools
  2. Open Terminal and run:
    adb version

Linux

  1. Many distros have it in their package manager:
    sudo apt install adb          # Debian / Ubuntu
sudo dnf install android-tools  # Fedora
  2. Or download directly from Google and extract.
  3. Confirm with:
    adb version

Part 4 — Connect the phone

  1. Use a USB-C data cable (not a charge-only cable — same plug, but cheap cables often skip the data wires).
  2. Plug the phone into the computer.
  3. On the phone, pull down the notification shade — you will see a USB connection notification. Tap it and choose File Transfer (MTP) or USB debugging. Either works for ADB.
  4. Run:
    adb devices
    • If this is your first connection you will see a prompt on the phone: "Allow USB debugging?". Tap Allow (optionally tick "Always allow from this computer").
    • Run adb devices again. You should see your device listed as device (not unauthorized).

Part 5 — Export the package list

Run one of the following commands. It will copy the output to your clipboard so you can paste it into the advisor.

Windows PowerShell

.\adb.exe shell pm list packages -f | Set-Clipboard

Or use the helper script from the repo:

.\scripts\export-packages.ps1

macOS / Linux

adb shell pm list packages -f | pbcopy    # macOS
adb shell pm list packages -f | xclip -sel clip  # Linux with xclip

Or use the repo script:

bash scripts/export-packages.sh

The output looks like:
package:/system/priv-app/FBServices/FBServices.apk=com.facebook.services
There will be a few hundred lines — that is normal.

Part 6 — Run the advisor

  1. Open the Package Advisor in your browser.
  2. Paste the clipboard contents into the text box.
  3. Click Analyze.
  4. Review the suggestions (green = keep, orange = review, red badge = suggested remove).
  5. Adjust checkboxes, then click Copy to clipboard.
  6. Back in your PowerShell / Terminal window, paste and run the commands (phone still connected).

Part 7 — After running commands

  • Test: make calls, check Messages, camera, Maps — verify nothing broke.
  • If something breaks, restore with:
    adb shell cmd package install-existing com.example.package
  • Turn USB debugging off again: Settings → Developer options → USB debugging → OFF.

Troubleshooting

Device not detected

  • Try a different USB cable (common cause).
  • Try a different USB port on the PC.
  • Windows: install Universal ADB Drivers.
  • Make sure USB debugging is still on (screen lock can toggle it off on some devices).

Still unauthorized after tapping Allow

  • Revoke all ADB authorizations: Settings → Developer options → Revoke USB debugging authorizations → then re-connect.

Failure [not installed for 0]

The package was already removed for this user or was never installed. Safe to ignore.
